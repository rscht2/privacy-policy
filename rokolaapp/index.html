Política de Privacidad

Rokola Digital es una aplicación de jukebox musical que te permite buscar y reproducir música conectándote a tu servidor Rokola personal. Respetamos tu privacidad y nos comprometemos a proteger tus datos personales.

1. Información que Recopilamos

1.1 Información que Proporcionas

  • Nombre de usuario: Para identificarte en la aplicación
  • Servidor URL: Para conectarte a tu servidor Rokola personal
  • Dedicatorias de canciones: Mensajes opcionales que añades a las canciones

1.2 Información Recopilada Automáticamente

  • Datos de uso: Firebase Analytics recopila información sobre cómo usas la app
  • Datos de crashes: Firebase Crashlytics recopila información sobre errores para mejorar la estabilidad
  • Datos de rendimiento: Firebase Performance Monitoring recopila métricas de rendimiento
  • Identificador de publicidad: Para mostrar anuncios personalizados (Google AdMob)

1.3 Permisos de la Aplicación

  • INTERNET: Para conectarse a tu servidor Rokola local y mostrar anuncios
  • ACCESS_NETWORK_STATE: Para verificar la conectividad de red
  • CAMERA: Para escanear códigos QR y conectarte rápidamente a servidores (opcional)

1.4 Servidor Local

  • La app se conecta a un servidor Rokola que TÚ instalas y controlas
  • El servidor se ejecuta en tu red local (LAN), no en internet
  • Tú configuras manualmente la dirección IP del servidor
  • Los datos de reproducción permanecen en tu red local

2. Cómo Usamos tu Información

  • Funcionalidad de la app: Para reproducir música y gestionar la cola de reproducción
  • Mejora del servicio: Analizar el uso y corregir errores
  • Publicidad: Mostrar anuncios relevantes mediante Google AdMob
  • Comunicación: Enviar notificaciones sobre el estado de reproducción

3. Compartir Información con Terceros

3.1 Servicios de Terceros que Usamos

  • Google Firebase: Analytics, Crashlytics, Performance Monitoring
  • Google AdMob: Servicio de publicidad
  • Google Play Services: Servicios básicos de Android
⚠️ Importante: No vendemos, alquilamos ni compartimos tu información personal con terceros para fines de marketing.

4. Seguridad de los Datos

4.1 Comunicación con Servidor Local

  • Tu servidor Rokola se ejecuta en tu red local privada (LAN)
  • Toda comunicación usa HTTPS (cifrado TLS/SSL)
  • El servidor usa certificados autofirmados para cifrado
  • La comunicación entre la app y tu servidor permanece dentro de tu red local
  • No transmitimos datos de tu servidor a través de internet público

4.2 Comunicación con Servicios de Google

  • Usamos HTTPS para todas las comunicaciones con servicios externos
  • Firebase (Analytics, Crashlytics, Performance)
  • Google AdMob
  • Google Play Services

4.3 Almacenamiento Local

  • Los datos se almacenan localmente en tu dispositivo
  • No almacenamos información personal en servidores externos
  • Puedes eliminar todos los datos desinstalando la app

5. Publicidad y Consentimiento

  • Usamos Google AdMob para mostrar anuncios
  • Implementamos User Messaging Platform (UMP) para cumplir con GDPR
  • Puedes gestionar tus preferencias de anuncios en la configuración de tu dispositivo

6. Tus Derechos

Tienes derecho a:

  • Acceder a tu información personal
  • Solicitar la corrección de datos incorrectos
  • Solicitar la eliminación de tus datos
  • Retirar el consentimiento para el procesamiento de datos

7. Retención de Datos

  • Los datos locales se mantienen hasta que desinstalas la app
  • Los datos de Firebase se retienen según las políticas de Google Firebase

8. Cumplimiento Legal

Esta aplicación cumple con:

  • GDPR (Reglamento General de Protección de Datos de la UE)
  • CCPA (Ley de Privacidad del Consumidor de California)
  • Políticas de Google Play Store

Seguridad y Cifrado

Conexión HTTPS con Certificados Autofirmados

Rokola Digital usa HTTPS para todas las conexiones al servidor local.

  • El servidor Rokola usa certificados autofirmados
  • La app acepta estos certificados para conexiones a redes privadas (LAN)
  • Toda comunicación está cifrada con TLS/SSL
  • Los datos no salen de tu red local

¿Por qué certificados autofirmados?

Los certificados autofirmados son seguros para servidores locales porque:

  • El servidor está en tu red privada (192.168.x.x, 10.x.x.x)
  • Tú configuras manualmente la dirección IP
  • Los datos permanecen en tu red local
  • TLS/SSL protege contra ataques en la LAN

Servicios Externos

Todas las comunicaciones con servicios de Google usan HTTPS con certificados válidos:

  • Firebase Analytics, Crashlytics, Performance
  • Google AdMob
  • Google Play Services
